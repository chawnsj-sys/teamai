# TeamAI — 前端交互规范

## 技术栈

- 纯 HTML + CSS + Vanilla JavaScript（无框架依赖）
- WebSocket 实时通信
- Markdown 渲染：marked.js
- 代码高亮：highlight.js
- 响应式布局：CSS Flexbox + Media Queries

## 核心组件

### 1. 频道列表（左侧栏）

- 分两组显示：群聊频道、私聊频道
- 每个频道项显示：频道名 + 未读消息数 + 最后消息时间
- 私聊频道显示对应 Agent 的 emoji 和名字
- 支持右键菜单：重命名、清空历史、删除
- 底部快捷入口：新建群聊、定时任务管理、全局设置

### 2. 消息流（主区域）

#### 消息气泡
- 用户消息：右对齐，蓝色背景
- Agent 消息：左对齐，灰色背景，显示 Agent 头像和名字
- 系统消息：居中，浅色文字（如"任务已分发"、"Agent 已加入"）

#### 消息内容渲染
- Markdown 完整支持（标题、列表、表格、引用、链接）
- 代码块语法高亮（支持 SQL、Python、JSON、JavaScript 等）
- 图片内联显示（支持点击放大）
- 表格响应式滚动

#### 元信息栏
每条 Agent 回复底部显示一行元信息：
```
🔍 记忆×3 | 📚 知识库×2 | 🔧 工具×5 | ⏱ 12s
📊 input:150 output:800 cacheRead:5000 total:5950
```
- 记忆数：本次检索命中的 AgentCore Memory 条数
- 知识库数：本次检索命中的 KB 文档数
- 工具数：本次调用的工具/Skill 次数
- 耗时：从发送到回复完成的秒数
- Token 统计：input/output/cacheRead/total
- 点击可展开查看具体的记忆内容和知识库片段

#### 流式输出
- Agent 回复通过 WebSocket 逐字流式显示
- 显示状态指示器：thinking → working → 完成
- 工具调用时显示工具名和执行状态

### 3. 输入区域

- 多行文本输入框（自动扩展高度，最大 6 行）
- @mention 自动补全（输入 @ 弹出 Agent 列表）
- 文件上传按钮（支持拖拽上传）
- 发送按钮（Enter 发送，Shift+Enter 换行）
- 群聊模式切换：隐私模式 / 开放模式

### 4. Agent 详情面板（右侧栏）

点击 Agent 头像或名字展开，包含多个 Tab：

#### 概览 Tab
- Agent 头像、名字、角色描述
- 当前状态（online / thinking / working / offline）
- 绑定的模型名称

#### 设置 Tab
- SOUL（人设）编辑器：Markdown 文本域
- IDENTITY（身份）编辑器
- MEMORY.md（精选记忆）编辑器
- 模型选择下拉框
- Skill 启用/禁用开关列表

#### 知识库 Tab
- 文件列表（文件名 + 大小 + 上传时间）
- 上传按钮
- 删除按钮（带确认）
- 搜索框（语义搜索知识库内容）

#### 记忆 Tab
- 记忆列表（按时间倒序）
- 每条记忆显示：内容摘要 + 创建时间
- 支持编辑、删除单条记忆
- 支持手动创建记忆
- 搜索框

### 5. PM 任务看板

当 PM 分发跨领域任务时，显示任务执行面板：

```
┌─────────────────────────────────────┐
│  📋 任务：分析 Q1 营销效果和用户行为  │
├─────────────────────────────────────┤
│  ✅ Nova (Amazon)    │ 12s │ 完成   │
│     → 用户行为数据分析               │
│  🔄 凌 (Snowflake)   │ 8s  │ 执行中 │
│     → 营销 ROI 分析                 │
│  ⏳ PM (Alex)        │ -   │ 等待   │
│     → 汇总报告                      │
├─────────────────────────────────────┤
│  [取消任务]                          │
└─────────────────────────────────────┘
```

- 实时更新每个子任务的状态和耗时
- 子任务完成后显示结果摘要
- 支持中途取消

### 6. 定时任务管理

独立页面或弹窗，列表展示所有定时任务：

| 字段 | 说明 |
|------|------|
| 任务名 | 可编辑 |
| Cron 表达式 | 支持可视化选择（每天/每小时/自定义） |
| 目标 Agent | 下拉选择 |
| 消息内容 | 发送给 Agent 的指令 |
| 状态 | 启用/禁用开关 |
| 上次执行 | 时间 + 结果（成功/失败） |
| 操作 | 手动触发、编辑、删除 |

### 7. 全局设置

- Gateway 连接状态
- Web 认证配置
- Agent 创建/删除
- 全局 SOUL 编辑（GLOBAL_SOUL.md）

## 交互流程

### 普通对话
1. 用户在输入框输入消息，点击发送
2. 前端通过 HTTP POST `/api/send` 发送消息
3. 服务端并行检索 Memory + KB，拼接上下文
4. 通过 WebSocket 将消息转发给 OpenClaw Gateway
5. Gateway 流式返回 Agent 回复
6. 前端通过 WebSocket 接收并逐字渲染
7. 回复完成后显示元信息栏

### PM 跨领域任务
1. 用户在群聊 @PM 提出跨领域问题
2. PM 返回 JSON 任务分配指令
3. 前端解析 JSON，显示任务看板
4. 系统并行将子任务分发给各专家
5. 每个专家的回复实时更新到任务看板
6. 所有子任务完成后，结果打包发给 PM
7. PM 生成汇总报告，显示在聊天区域

### Agent 间重定向
1. 专家 Agent 判断问题不在自己领域
2. 返回 JSON：`{"action": "redirect", "reason": "...", "suggest": "agent-id"}`
3. 前端显示重定向提示
4. 系统自动将问题转发给建议的 Agent

## 响应式设计

### 桌面端（>1024px）
- 三栏布局：左侧栏(240px) + 主区域(flex) + 右侧栏(320px)
- 右侧栏默认收起，点击展开

### 平板端（768px - 1024px）
- 两栏布局：左侧栏可收起 + 主区域
- 右侧栏改为浮层弹出

### 移动端（<768px）
- 单栏布局
- 频道列表、聊天、详情面板通过导航切换
- 输入框固定在底部

## 视觉规范

### 配色
- 主色：#2563EB（蓝色，用于按钮、链接、用户消息背景）
- 背景：#F8FAFC（浅灰）
- 卡片：#FFFFFF
- 文字：#1E293B（主文字）、#64748B（次要文字）
- 成功：#10B981
- 警告：#F59E0B

### 字体
- 正文：system-ui, -apple-system, sans-serif（14px）
- 代码：'SF Mono', 'Fira Code', monospace（13px）
- 标题：16px / 18px / 20px

### 间距
- 组件间距：16px
- 内边距：12px - 16px
- 消息间距：8px
- 圆角：8px（卡片）、4px（按钮）
